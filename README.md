# Brand Kit Generator

> A system prompt that turns LLMs into opinionated brand strategists.
> Refuses to write generic kits. Refuses to ship until the kit is a
> decision tool, not decoration.

---

## What this is

`prompt.md` is a 600-line system prompt that takes a founder through an
8-stage interview and produces a brand kit you can actually use to make
daily decisions — what projects to refuse, what copy to write, what to
delete from the homepage.

The prompt is reverse-engineered from a real brand kit
([acelera-agency/brand](https://github.com/acelera-agency/brand)) — not
generated by guessing what brand kits "should" look like.

## What this is **not**

- **Not "AI brand kit in 5 minutes".** The interview takes 60-90 minutes.
  If you want a 5-minute kit, this tool will refuse you.
- **Not a logo generator.** It defines the *concept* a designer should
  execute. It doesn't draw artwork.
- **Not a moodboard generator.** It forces you to commit to constraints
  (palette tokens with roles, type families with reasons), not vibes.
- **Not generic.** The output of this prompt cannot be re-pasted on
  another company's homepage. If it could, the prompt failed.

---

## Why it's different

The prompt encodes 8 non-negotiable beliefs. Read them once before using:

1. **Every brand must name an enemy.** Not a competitor — a *practice* the
   brand opposes. Without an enemy there is no tension; without tension
   there is no position.
2. **Anti-positioning before positioning.** You haven't earned your
   positioning until you've named at least 5 things you'll lose business by
   refusing to do.
3. **The three-layer stack: Character / Promise / Method.** These do
   different jobs. Most kits collapse them into one paragraph and nothing
   lands.
4. **Voice as constraints, not adjectives.** "Friendly and professional"
   tells a writer nothing. The voice section produces a do/don't list with
   *real phrases* a stranger could write a tweet from.
5. **Application > theory.** Every kit ends with templates the brand will
   actually paste into surfaces — hero copy, cold email body, first-minute
   meeting script — using the brand's real working phrases.
6. **Surface-specific non-negotiable rules.** Outreach has different rules
   than meetings has different rules than proposals. Each rule has the form
   *rule + the one-line reason it matters*.
7. **Visual restraint as positioning.** Force a choice: refined consultancy
   *or* maximalist editorial. Hybrid is forbidden.
8. **The kit is a decision tool, not a deck.** It's done when a new hire
   can write outreach in the brand voice on day one without asking.

These beliefs are inflexible. The generator will refuse to advance through
the interview until each belief is satisfied.

## Three ways to use it

### 1. As a Claude / ChatGPT system prompt

Paste the contents of [`prompt.md`](./prompt.md) as the system message of a
new chat with Claude (Sonnet/Opus) or ChatGPT (GPT-4 or better). Send any
greeting and the generator will start the calibration question.

```
System: <paste prompt.md>
You: hi
```

### 2. As a Claude Code skill

Clone this repo into a Claude Code workspace and tell Claude to run the
brand kit generator using `prompt.md` as its system prompt. Works as either
a one-off conversation or an automated subagent.

### 3. As a manual for human strategists

Read [`prompt.md`](./prompt.md) end-to-end. The 8 stages, the pushback
rules, and the gates are usable by a human running an actual workshop —
no LLM required. The methodology is the product; the prompt is one
delivery format.

---

## The 8-stage interview

Each stage has a goal, questions to ask one at a time, pushback rules for
vague answers, and a **gate** the conversation must pass before advancing.

| # | Stage | Gate |
|---|---|---|
| 0 | Context & contradiction | Write a 3-sentence "before vs after" that sounds like the founder, not a press release |
| 1 | The enemy | Founder names the enemy in one sentence; another team member would recognize it |
| 2 | Three-layer stack (Character / Promise / Method) | Three working phrases, each under 10 words, each distinguishable |
| 3 | Anti-positioning | At least 5 lines, each costing the brand something concrete |
| 4 | ICP by signals | 4-6 behavioral signals observable in the first 10 minutes of a sales call |
| 5 | Voice as constraints | A new hire could write a tweet in-brand from this section alone |
| 6 | Application templates | Each template usable the same day on a real surface |
| 7 | Visual direction | A designer could mock up a homepage that's recognizably *this brand* on the first try |
| 8 | Non-negotiable rules per surface | A reviewer can grade any new piece against the rules in 60 seconds |

## Forbidden words

The prompt enforces a list of words it refuses to produce. If you see any
of these in a generated kit, file an issue:

> *innovation, disruption, transformation, next-generation, cutting-edge,
> seamless, holistic, end-to-end, empowering, revolutionary, unleash,
> unlock, supercharge*

These are signals of genericness. The prompt replaces them with concrete
claims, every time.

## Contributing

Contributions welcome — but with the same standard the generator holds you
to. Specifically:

- **Calibration examples from other real brands** that pass the standard
  (not invented). PRs adding a new section to the calibration with
  attribution are great.
- **Stage refinements** — but include the rationale: what edge case were
  you fixing? What gate wasn't strict enough?
- **Translations of the prompt itself** (the methodology stays in English,
  but the prompt can be translated into other languages for non-English
  founders).
- **Bug reports on edge cases** the prompt mishandles. Concrete inputs +
  what the generator did wrong + what it should have done.

What's **not** welcome:

- "Make it less opinionated" — the opinions are the product.
- "Add more options" — every option is a degraded gate.
- "Soften the pushback rules" — vagueness is what it's there to defeat.
